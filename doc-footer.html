Suchen

By ID
In alle endpoints kann gesucht werden und zwar nach ID: Beispiel "https://url/endpoint/34" sucht nach endpoint mit id gleich '34'. Resultat bei gefunden 200, bei nicht gefunden 404. Bei Erfolg ist das Resultat in JSON Objekt
By Example
Auf alle andere Attribute kann gesucht werden mit query-string. Beispiel "https://url/endpoint/?title=hello". Gross-/Kleinschreiben bei Key (title) beachten, beim Wert (helloe) spielt is keine Rolle. Auch Jokerzeichen sind möglich "https://url/endpoint/?title=h*" sucht alle endpoints mit Feld title beginnend mit 'h'. Bei Erfolg (200) ist das Resultat immer ein JSON Array von Objekte. Resultat bei Key nicht gefunden (missspelled): 400 Bad Request, bei nicht gefunden 204 no content

Methoden

Die folgenden Methoden sind zulässig:

GET
Daten für Endpunkt abrufen
POST
Erstellen Sie eine neue Entität. Der Text enthält vollständige Details.
PUT
Vorhandenen Eintrag für Endpunkt aktualisieren. Denken Sie daran, /{id} einzuschließen, der Inhalt sind die geänderte Felder.
LÖSCHEN
Vorhandenen Eintrag für Endpunkt entfernen. Denken Sie daran, /{id}
anzugeben

Statuscodes

Der Dienst folgt REST-Standards mit den folgenden HTTP_Statuscodes:

200
Erfolg
204
Erfolg, aber keine Ergebnisse.
400
Ungültige Anfrage. Der Text enthält Details.
404
Endpunkt nicht gefunden. Inhalt leer
500
Interner Serverfehler. Der Text enthält Details.
